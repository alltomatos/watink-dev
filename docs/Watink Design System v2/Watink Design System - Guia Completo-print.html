🎨 Watink Design System

Guia Completo de Temas, Cores, Tipografia e Componentes

Plataforma open-source brasileira de atendimento multiagente no WhatsApp

📋 Visão Geral

O Watink é uma plataforma que centraliza o atendimento via WhatsApp para equipes. Este design system fornece diretrizes visuais, tokens de design e componentes reutilizáveis para manter consistência em todas as interfaces.

🎯 Propósito

Garantir consistência visual e experiência uniforme em toda a plataforma

🌐 Multi-tema

Suporte para temas: Default, Google, Apple, WhatsApp e SaaS

🌙 Dark Mode

Modo escuro completo com data-theme="dark"

📱 Responsivo

Design agnóstico ao framework usando CSS custom properties

📐 Padrões de Layout

Sidebar

  • Expandida: 260px — com ícone + label
  • Recolhida: 72px — somente ícone
  • Fundo: --bg-sidebar: #1E293B (slate-800)
  • Comportamento: Recolhe para trilho em mobile e por padrão no desktop

Barra de Aplicativo (AppBar)

  • Altura: 64px
  • Superfície: Branco (--bg-appbar)
  • Borda inferior: 1px --border-subtle
  • Sombra: Nenhuma por padrão

Cards

  • Border Radius: 16px (base), 20px (modais)
  • Sombra: 0px 4px 20px rgba(0,0,0,0.08) — suave, difusa
  • Sem borda visível — profundidade apenas por sombra
  • Hover: translateY(-6px) + sombra mais profunda

✨ Animação e Movimento

Elemento Transição Duração Easing
Botões — Hover translateY(-1px) + glow 150ms ease-out
Cards — Hover translateY(-6px) 300ms spring
Nav items — Hover x: 4px 200ms ease-out
Nav items — Press scale: 0.98 150ms ease-out

Regra: Nunca usar apenas opacidade para hover — sempre incluir uma mudança de cor de fundo ou transform.

🌙 Modo Escuro

O modo escuro é ativado adicionando data-theme="dark" ao elemento <html>. Todas as variáveis CSS são atualizadas automaticamente.

Token Modo Claro Modo Escuro
--bg-default slate-50 neutral-950
--bg-surface white slate-800
--text-primary neutral-900 slate-100
--action-primary blue-500 blue-400

As sombras também são ajustadas no modo escuro para manter a hierarquia de profundidade — são elevadas para compensar backgrounds mais escuros.

📋 Melhores Práticas

✅ Faça

  • Use variáveis CSS para cores — var(--action-primary)
  • Aplique hover sempre com transform + cor
  • Use --shadow-md para cards, --shadow-strong para modais
  • Mantenha tamanho mínimo de texto em 15px (--text-body)
  • Use cores de status primárias: sucesso, erro, aviso, info
  • Agrupe elementos com display: flex ou grid + gap

❌ Não faça

  • Não use hex direto — use tokens CSS
  • Não aplique cores Google fora da sidebar (tema google)
  • Não reutilize verde WhatsApp para UI genérica
  • Não crie componentes com hover somente por opacidade
  • Não use tamanhos de fonte menores que 12px em interface
  • Não crie novas cores — use a paleta existente
